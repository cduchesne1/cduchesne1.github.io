JigSwap, une application d'échange de casse-têtes

Ce projet, est une application mobile codée avec Flutter pour un client de Vancouver. Cette application permet de trouver des personnes désirant échanger un casse-tête près de leur position. Il est possible de rechercher les casse-têtes offerts selon la position, la catégorie, le nombre de pièces ou le nom. Une fois la requête d'échange envoyée, les utilisateurs peuvent converser entre-eux afin de déterminer un lieu de rencontre pour échanger les casse-têtes.

01. Technologies employées

Ce projet a été réalisé avec Flutter. Flutter est un framework développé par Google et basé sur le langage Dart. L'éditeur utilisé afin de coder cette application était Visual Studio Code. XCode ainsi qu'Android Studio ont aussi été utilisés afin d'effectuer le packaging pour les différentes plateformes. Adobe XD a été utilisé afin de modifier l'interface graphique. Finalement, Firebase a été utilisé afin de concevoir le back-end de l'application.

02. Interface graphique adaptative

L'interface graphique de ce projet a été implémentée de façon à ce qu'elle soit jolie sur toutes les dimensions d'écrans à l'exception des tablettes.

03. Firebase

Différents services de Firebase ont été utilisés afin de rendre l'application fonctionnelle. En effet, Firebase Auth a été utilisé pour l'authentification des utilisateurs, Cloud Firestore a été utilisé afin de stocker les données des utilisteurs et Firebase Storage a été utilisé afin de stocker des fichiers tels des images. Il faut noter que Cloud Firestore est une base de données de type NoSQL qui stocke les données sous forme de documents. Cloud Messaging ansi que Cloud Functions ont été utilisés afin d'envoyer des notifications aux utilisateurs. Google ML Kit a été utilisé afin de s'assurer que les photos soumises par les utilisateurs sont bels et bien des photos de casse-têtes, l'API Google Places a été utilisé afin de permettre la recherche d'endroits, Crashlytics a été utilisé afin de monitorer les éventuels problèmes et App Distribution a été utilisé afin de distribuer des versions de test de l'application. De plus, Google AdSense a été utilisé afin de présenter des annonces aux utilisateurs.

04. Sécurité

Puisque l'application stocke les données des utilisateurs dans une base de données, des règles de sécurité ont été implémentées autour de la base de données afin que seuls les utilisateurs autorisés puissent accéder à l'information.
