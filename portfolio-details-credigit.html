Credigit

Ce projet a été le projet final de mon diplôme d'études collégiales. Nous avons décidé de créer un système de paiement par empreinte digitale. Nous avons créé un sytème d'inscription, un point de vente, un terminal de paiement ainsi qu'un tableau de bord afin d'offrir une solution clé en mains pour les commerçants désirant utiliser notre technologie. Afin d'assurer la sécurité des données, nous avons implémenté deux algorithmes d'encodage: AES et RSA. Le tout a été codé en Java et nous avons utilisé un Raspberry Pi afin de créer le terminal de paiement. Il faut noter que la compagnie Bank-Era Corp., mentionnée dans la vidéo, est une entreprise fictive et que le projet appartient à ses créateurs uniquement.

Détails

01. Technologies employées

Ce projet a été réalisé en Java. L'éditeur utilisé pour créer ce projet était Eclipse. Afin de réaliser l'interface graphique, nous avons utilisé Scene Builder. Afin de modéliser l'interface graphique, nous avons utilisé Adobe XD. Un Raspberry Pi, une LED, un boîtier en plastique ainsi qu'un lecteur d'empreintes digitales ont été utilisés afin de fabriquer le terminal. Finalement, MongoDB a été utilisé comme base de données pour contenir les informations des établissements et des utilisateurs.

02. Architecture MVC et tests unitaires

L'architecture MVC (modèle, vue, contrôleur) a été employée afin de bien séparer la logique du visuel. De plus, des tests unitaires de type JUnit ont été implémentés afin d'assurer la qualité du code.

03. Base de données

Nos applications étaient reliées à une base de données afin de maintenir les informations des utilisateurs à jour. Dans le cadre de ce projet, nous avons utilisé MongoDB qui est une base de données de type NoSQL stockant les données sous forme de documents JSON.

04. Encodage

Puisque notre projet visait des informations sensibles, nous avons implémenté des algorithmes d'encodage afin de renforcer la sécurité. Nous avons utilisé AES (Advanced Encryption Standard) afin d'encoder toutes les données et nous avons utilisé RSA afin d'encoder la clé de chiffrement de l'algorithme d'AES.

05. Terminal de paiement

Afin de fabriquer le terminal de paiement, nous avons connecté un lecteur d'empreintes digitales à un Raspberry Pi. Nous avons utilisé le Raspberry Pi, car celui-ci supporte les applications Java.

06. Point de vente

Puisqu'il n'existe aucun point de vente supportant le nouveau terminal que nous avons créé, nous avons créé notre propre point de vente afin que les entreprises puissent utiliser notre terminal. Le point de vente communique avec le terminal via Wi-Fi.

Compétences acquises

01. Méthodologie Agile

Afin de mettre ce projet à terme, nous avons été contraints d'utiliser la méthodologie Agile. En effet, nous avons créé un dossier de conception comportant des User Stories, nous avons effectué des Sprints, des rapports de sprint et des Daily Scrums.

02. Conception

Ce projet a été très formateur au niveau de la conception d'un produit. En effet, nous devions nous assurer que notre produit comblait un réel problème, que les fonctionnalités implémentées répondaient aux besoins des différents utilisateurs et que notre programme était facile d'utilisation.

03. Structure Serveur/Client

Afin que le terminal communique avec le point de vente, nous devions faire en sorte que le terminal agisse comme un serveur et que le point de vente agisse comme un client se connectant au terminal. Nous n'avions jamais effectué quelque chose du genre et donc nous avons été contraints d'apprendre comment le faire par nous-mêmes.

04. Algorithmes d'encodage

Pour protéger nos données, nous avons décidé d'utiliser l'AES et le RSA. Nous avons donc été contraints d'apprendre comment ces algorithmes fonctionnent et de trouver une façon de les implémenter en Java.
